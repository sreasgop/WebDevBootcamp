Basics of HTML


What is HTML?

HTML stands for Hypertext Markup Language.

  • Hypertext refers to the way web pages are organized, with text that can be linked to other pages or resources. This allows users to navigate between related pages on the web by clicking on hyperlinks.
  • Markup Language refers to the way HTML documents are formatted using tags and attributes that define the structure and content of the page. HTML tags are used to specify different types of content, such as headings, paragraphs, images, links, and tables, while attributes provide additional information about the tags, such as the source of an image or the target of a hyperlink.

Together, Hypertext and Markup Language form the basis of HTML, which is the standard language used to create web pages. HTML provides a set of rules and conventions that enable developers to create web pages that can be displayed consistently across different devices and platforms. HTML documents are interpreted by web browsers, which render the content of the page on the user's screen, allowing them to interact with the page and access the resources and information it contains.



History of HTML

The history of HTML begins in the late 1980s, when a researcher at the European Organization for Nuclear Research (CERN) named Tim Berners-Lee began working on a project to create a way for scientists to share and collaborate on research documents over the Internet. Berners-Lee recognized the need for a way to mark up text with formatting instructions so that it could be displayed consistently across different computer systems.
In 1989, Berners-Lee proposed a system for creating hypertext documents called "WorldWideWeb" (later renamed to "Nexus"). This system used a markup language called "HTML" to define the structure and content of web pages, and a web browser called "WorldWideWeb" to display them.
The first version of HTML, HTML 1.0, was released in 1993, and included basic tags for formatting text and creating links. Over the next few years, new versions of HTML were released, including HTML 2.0 in 1995 and HTML 3.2 in 1997, which added new features like tables, forms, and image maps.
In 1999, the World Wide Web Consortium (W3C) released HTML 4.01, which included new features like frames, style sheets, and scripting. HTML 4.01 was the last major version of HTML to be released before the development of XHTML, a reformulation of HTML as an XML application, which was released in 2000.
In 2014, the W3C released the first draft of HTML5, a major revision of HTML that introduced new semantic tags, multimedia support, and improved accessibility features. HTML5 has since become the de facto standard for web development, and is widely used to create modern, interactive, and responsive web pages.



What is an HTML element?

HTML (Hypertext Markup Language) elements are the building blocks of web pages. An HTML element is defined by a start tag, some content, and an end tag. The content between the start and end tags is what is displayed on the web page.



What is an HTML attribute?

HTML elements can also have attributes that provide additional information about the element, such as the ID, class, and style. An HTML attribute is a property or characteristic that is used to modify the behavior or appearance of an HTML element. Attributes are added to the opening tag of an HTML element, and they are written as key-value pairs, with the attribute name followed by an equals sign (=) and the attribute value surrounded by quotes (" "). HTML attributes can be used to accomplish a variety of tasks, such as:

  • All HTML elements can have attributes
  • Attributes provide additional information about elements
  • Attributes are always specified in the start tag
  • Attributes usually come in name/value pairs like: name="value"
  • Specifying the source of an image or other media file (using the "src" attribute)
  • Creating hyperlinks to other pages or resources (using the "href" attribute)
  • Defining the type of an input field in a form (using the "type" attribute)
  • Assigning a unique ID to an element for JavaScript manipulation (using the "id" attribute)
  • Adding descriptive text to an image for accessibility purposes (using the "alt" attribute)



What is the difference between HTML elements and HTML tags?

The terms "HTML element" and "HTML tag" are often used interchangeably, but they refer to slightly different things in HTML:

  • HTML element: An HTML element refers to a complete element, including both the opening tag, content, and the closing tag (if applicable), as well as any attributes or properties that apply to the element. Examples of HTML elements include <p>, <div>, and <img>.
  • HTML tag: An HTML tag refers specifically to the markup code used to define an element. A tag is a snippet of code surrounded by angle brackets (< and >), and it can include attributes that modify the behavior or appearance of the element. Examples of HTML tags <p>, <div>, and <img>.

In essence, an HTML tag is a part of an HTML element that defines the element's purpose and behavior, while an HTML element is the complete package, including the tag, content, and attributes. It's worth noting that the terms "HTML element" and "HTML tag" are often used interchangeably in casual conversation, and in many cases, the distinction between the two isn't important. However, understanding the difference between the two can help you write and debug HTML code more effectively.



Types of HTML tags:

In HTML, tags can be classified into two main types: container tags and empty tags.

  1. Container tags: These tags are also called "paired tags" or "start and end tags", as they require both an opening tag and a closing tag to define a block of content. Container tags are used to group content together, and the content between the opening and closing tags is affected by the tag's attributes and any CSS styles applied to it. Examples of container tags include <div>, <p>, <ul>, <table>, and <form>.
  2. Empty tags: These tags are also called "void tags" or "self-closing tags", as they do not require a closing tag and are used to insert elements or content that do not have any content or do not require a container. Empty tags typically have attributes that define their behavior, such as the source of an image or the type of an input field. Examples of empty tags include <img>, <br>, <input>, and <hr>.

It's important to note that some elements, such as <meta> and <link>, can be both empty and container tags depending on the context and the presence of attributes. Additionally, some elements, such as <button> and <textarea>, are container tags but do not require a closing tag in HTML5.



HTML Boilerplate

    HTML Boilerplate explanation

  • All html documents start with this code: <!DOCTYPE html> The following line tells the browser that it is a html document.
  • <html lang="en"> marks the beginning of the root element of the html document. All the other html elements is going to go within this root html element. The lang attribute of the root element defines the language of the content of the website. This is not very important for the users who can see the website, it is more important for the users who can't. It helps various screen readers and other assistant technologies that will look at this language attribute so that when they read the website they can pronounce the word correctly.
  • The head of the website is defined by the <head> tag. This is an area where important information about the website is placed that is not displayed to the user. This is the opening tag for the head section of the document. The head section contains meta information about the document, such as the title, character encoding, and other metadata.
    • <meta charset="UTF-8"> It ensures that the characters that we are going to use in our website is displayed correctly.
    • <meta http-equiv="X-UA-Compatible" content="IE=edge"> is used to set the compatibility mode for Internet Explorer. It instructs Internet Explorer to use the latest rendering engine available, rather than using an older version or compatibility mode. This is important because different versions of Internet Explorer can have different rendering engines, and using the latest one ensures that the website will display correctly.
    • <meta name="viewport" content="width=device-width, initial-scale=1.0"> is used to set the viewport of the website. The viewport is the visible area of the web page on a device's screen. By setting the width to device-width, the website will be scaled to fit the screen width of the device it is being viewed on. The initial-scale=1.0 sets the initial zoom level of the website to 100%, meaning it will display at its natural size on the device's screen. This helps ensure that the website is displayed optimally on a wide range of devices, from desktop computers to smartphones and tablets.
    • <title> tag in HTML is used to specify the title of a web page. It is placed inside the <head> section of an HTML document, and the content inside the tag is displayed in the browser's title bar. The title tag is also used by search engines to identify the content of the page.
  • <body> tag is the opening tag for the body section of the document. The body section contains the actual content of the web page, such as text, images, and other media.
